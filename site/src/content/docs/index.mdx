`vpn-confinement` provides fail-closed WireGuard confinement for selected NixOS
systemd services. It is designed for deployments where some workloads should use
the tunnel while the host and other services stay on normal networking.

## When to use it

Use `vpn-confinement` when you want to:

- confine only the services that should use VPN egress
- keep host networking unchanged for non-confined workloads
- apply DNS and firewall policy at the namespace boundary
- tear services down when the namespace or WireGuard path disappears

## Security characteristics

- The trust boundary is the namespace, not the individual service.
- Confinement uses namespace-local nftables policy and generated resolver files.
- `dns.mode = "strict"` is intended to reduce classic resolver leak paths, not
  to guarantee that all DNS exfiltration techniques are impossible.
- `securityProfile = "highAssurance"` is the stricter mode for
  destination-constrained egress and stronger assertions.

## Typical setup

1. Import the module and enable `services.vpnConfinement`.
2. Define a namespace with a WireGuard interface and DNS policy.
3. Opt specific services or sockets into that namespace.

Use the [`Generated Options`](./reference/options-generated/) reference for
exact option names, defaults, and configuration details.
