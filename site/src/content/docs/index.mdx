<section class="flex flex-col gap-4">
  <h2>Why this exists</h2>
  <p class="m-0 max-w-[44rem] opacity-80">
    <code>vpn-confinement</code> is built for the common NixOS problem where
    only a handful of services should use a VPN, while the host and the rest of
    the system should stay on the normal network.
  </p>
  <ul class="m-0 pl-5">
    <li>Confine only the units that need tunnel egress.</li>
    <li>
      Keep policy at the namespace boundary instead of scattering it per
      service.
    </li>
    <li>
      Prefer fail-closed teardown when the namespace or tunnel disappears.
    </li>
    <li>
      Treat strict DNS as resolver leak resistance, not a blanket exfiltration
      claim.
    </li>
  </ul>
</section>
